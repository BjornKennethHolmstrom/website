#### **for [TOMLAB](https://tomopt.com/tomlab/)**

* * *

##### **GOSitS**

GOSitS is an algebraic/symbolic solver for global optimization solver written in object-oriented MATLAB code. It has not reached release status as of date.

* * *

##### **tomEval**

tomEval compares and evaluates optimization solver performances over standard test collections. Written in object-oriented [**MATLAB**](https://www.google.com/url?sa=t&rct=j&q=&esrc=s&source=web&cd=1&cad=rja&uact=8&ved=2ahUKEwiD8YjEmIneAhWGBywKHV9EC04QFjAAegQIAxAC&url=https%3A%2F%2Fen.wikipedia.org%2Fwiki%2FMATLAB&usg=AOvVaw0QW0HE1Qs8SWaldOALsGEc) code the user is able to select solvers and problem collections, run tests and view the results with a few simple commands.

* * *

##### **[ecpMINLP](https://tomopt.com/tomlab/products/mipnlp/solvers/ecpMINLP.php) and** [**stoaMINLP**](https://tomopt.com/tomlab/products/mipnlp/solvers/stoaMINLP.php)

ecpMINLP and stoaMINLP are two solvers for [**mixed integer nonlinear programming**.](https://glossary.informs.org/ver2/mpgwiki/index.php?title=Mixed-integer_program) They are written in standard procedural [MATLAB](https://www.google.com/url?sa=t&rct=j&q=&esrc=s&source=web&cd=1&cad=rja&uact=8&ved=2ahUKEwiD8YjEmIneAhWGBywKHV9EC04QFjAAegQIAxAC&url=https%3A%2F%2Fen.wikipedia.org%2Fwiki%2FMATLAB&usg=AOvVaw0QW0HE1Qs8SWaldOALsGEc) code. The solvers take advantage of other state-of-the-art solvers for [nonlinear](https://www.google.com/url?sa=t&rct=j&q=&esrc=s&source=web&cd=1&cad=rja&uact=8&ved=2ahUKEwjRvN_R-4jeAhWFVSwKHQXrAmwQFjAAegQICRAB&url=https%3A%2F%2Fen.wikipedia.org%2Fwiki%2FNonlinear_programming&usg=AOvVaw3DMvnIkeiVrSU-sVpSHOwR) and [linear programming](https://www.google.com/url?sa=t&rct=j&q=&esrc=s&source=web&cd=1&cad=rja&uact=8&ved=2ahUKEwjiuenG-4jeAhUwiKYKHQYJBMAQFjAAegQIBBAC&url=https%3A%2F%2Fen.wikipedia.org%2Fwiki%2FLinear_programming&usg=AOvVaw1fsREbheIDh8ZJ_Vhd3uvA) available within the optimization environment [TOMLAB](https://www.google.com/url?sa=t&rct=j&q=&esrc=s&source=web&cd=1&cad=rja&uact=8&ved=2ahUKEwil3I7RmIneAhUBWSwKHZxbDpEQFjAAegQIChAB&url=https%3A%2F%2Fen.wikipedia.org%2Fwiki%2FTOMLAB&usg=AOvVaw3u6tLFLxqb0elFZASTBHgp) and are sold as part of the solver package [/**MIPNLP**](https://tomopt.com/tomlab/products/mipnlp/).

#### **for Stampy**

### ADIT

**A**udio-based **D**igital **I**nformation **T**ransfer. A protocol for sending a short data sequence of a few bytes as high-frequency sounds between mobile devices. It was partly developed in Matlab, then implemented in Java and reached a fairly workable status. An unsolved problem was that the sound was still audible, contrary to the goal of the project. The code was never included in any finished product.
